# TryHackMe Write-ups

A collection of my write-ups from TryHackMe labs, documenting my hands-on practice in offensive and defensive security. Each write-up follows a consistent methodology — reconnaissance, exploitation, and remediation - reflecting how I approach real-world engagements.

## About This Repository

This repo serves as a running log of my practical cybersecurity training across various TryHackMe learning paths and rooms, including (but not limited to) web application penetration testing, infrastructure exploitation, cloud security, and the OWASP Top 10.

Rather than listing specific completed rooms here (since that list grows constantly), write-ups are organized by folder/file so you can browse what's currently documented directly in the repo structure.

## Structure

Each write-up generally follows this format:

- **Reconnaissance / Enumeration** — mapping the attack surface.
- **Vulnerability Identification** — pinpointing the flaw and how it was discovered.
- **Exploitation** — step-by-step walkthrough with commands and evidence.
- **Remediation** — practical fixes an engineering team could implement.

This mirrors the structure expected in real penetration test reports: technical depth for engineers, paired with clear, actionable recommendations for decision-makers.

## What This Demonstrates

- **Methodical approach** — enumeration before exploitation, always.
- **Root-cause thinking** — not just "how it broke," but *why* it broke and how to fix it.
- **Clear communication** — technical findings written so both engineers and non-technical stakeholders can act on them.
- **Continuous learning** — staying current with modern threats, including AI/LLM-specific attack surfaces (prompt injection, insecure AI design) alongside classic OWASP categories.
- **Ethical practice** — all testing performed exclusively in authorized, isolated lab environments provided by TryHackMe.

## Disclaimer

All content in this repository was obtained through legal, authorized lab environments (TryHackMe) intended for educational purposes. None of the techniques, credentials, or findings documented here apply to real, production, or third-party systems.
